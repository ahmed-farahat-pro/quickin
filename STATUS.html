3 What's done — in detail

Backend & database

  • Next.js API + node‑postgres on Neon — no Supabase, deployed to Vercel.
  • Email + password auth with 6‑digit OTP email verification (your Private Email SMTP).
  • Three roles — guest · host · admin; hardcoded admin login.
  • Google sign‑in verification (web + iOS audiences) · Apple token verification.
  • Reservation lifecycle — host listings, booking pending → confirm/reject, reservation codes; authorization enforced.

Web (quickin‑frontend)

  • Browse, live search (location/dates/guests + availability), Leaflet map with price pins.
  • Login/Sign‑up with Guest/Host choice + OTP screen; admin dashboard.
  • /host — add a listing, your listings, reservation requests with confirm/reject.
  • /reservation/[id] — details + on‑page QR; "Add to Apple Wallet" button present.
  • Google button live (with key); Apple button wired, hidden until configured.

iOS (SwiftUI)

  • Zoom splash, browse‑before‑login, search, MapKit price‑pin map, branded date picker.
  • Auth + OTP + roles; native Sign in with Apple & Google code.
  • Host dashboard + reservation detail with a CoreImage QR.
  • Bundle com.quickin.ahmed, v1.0.0 — archive verified, TestFlight‑ready.

Android (Jetpack Compose)

  • Browse, search, osmdroid price‑pin map, custom date picker.
  • Auth + OTP + roles; Google sign‑in scaffolding.
  • Host dashboard + reservation in‑app card with QR (ZXing) — not Apple Wallet, by design.
  • Signed release build config; both debug & release compile.

5 How to run everything

The web + backend are already live on Vercel. To run locally or the mobile apps:

Web & backend (live)

# already deployed — just open:
Web      https://quickin-frontend.vercel.app
Backend  https://quickin-backend.vercel.app
Admin    log in as  admin / (admin password)

Backend — local

cd ~/Downloads/quickin-backend
npm install
cp .env.example .env   # fill DATABASE_URL, AUTH_SECRET, SMTP_*
npm run dev            # http://localhost:4000

Web — local

cd ~/Downloads/quickin-frontend
npm install
# .env.local: NEXT_PUBLIC_API_URL=https://quickin-backend.vercel.app
npm run dev            # http://localhost:3000

iOS — simulator

cd ~/Downloads/quickin-master/mobile/ios
xcodegen generate
open QuickIn.xcodeproj # then press Run ▶

Android — emulator

open ~/Downloads/quickin-master/mobile/android
# in Android Studio, press Run ▶  — or:
./gradlew :app:installDebug

Try the live flow (no device)

  • Sign up as Host → add a listing.
  • Private window → sign up as Guest → book it.
  • Back as host → confirm → guest sees the QR.
